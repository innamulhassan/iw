02Anatomy of a case

Each case shares the same shape — the differential-diagnosis discipline is what makes the pairs genuinely distinct rather than a re-skinned template.

INC-7734

orders-api latency after an index drop

Database✓ goldenlive
origin
ServiceNow incident INC-7734; CHG-9 (a DB migration by dba-jsmith) landed 8m before onset.
symptom
orders-api p99 spikes to 5.2s while p50 stays flat at 46ms and errors at 1.5% — the tail alone is blown.
topology
service → database (JDBC exit-call to orders-pg); the endpoint reading order_items is the one on fire.
changes
CHG-9 dropped idx_order_items_order_id.
root cause
change_event:chg-9 — missing index → full-table scans → JDBC pool exhaustion.
tools
appd.get_snapshots (exit-call) · prometheus.instant_query (pool 200/200) · servicenow.find_recent_changes · git.diff_range (the DROP INDEX line)
Discriminator vs. code regression: the service's own compute (p50) is unaffected; the tail is entirely JDBC-wait-bound. That flat p50 refutes an application code path.
INC-9001

checkout → pricing timeouts after an MTU change

Network✓ goldenlive
symptom
Cross-service timeouts ~30m after an edge MTU/uplink change; retransmits climb on the segment.
topology
checkout-svc → pricing-svc across network segment SEG-EDGE-12; pricing-db ruled out.
root cause
network_segment:seg-edge-12 — wire-level loss at the segment boundary.
tools
prometheus.fetch_metrics (retrans_segs, probe_success) · appd.flowmap (exit-call to pricing) · servicenow.find_recent_changes
Discriminator: the caller's own p50 is fast and the callee DB pool is healthy (24% util) — the loss is on the segment, not either endpoint.
INC-4821

payments-api 5xx after v4.12.0

Application code✓ goldenlive
symptom
5xx error rate jumps; a NullPointerException in TaxCalculator.java:88.
topology
service → database (ruled out); pods stay Ready.
root cause
code_commit:abc123 — getRegion() is null for intl orders → NPE.
tools
appd.bt_health / fetch_traces · git.blame + diff_range + get_pr_for_commit · splunk.error_signature_topk
Discriminator: a refuted_variant forces a backtrack — the DB pool is healthy, so the fault is the code path v4.12.0 introduced, blamed to file:line.
INC-7731

checkout-api CrashLoopBackOff after rev43

Deployment✓ golden
symptom
Pods never reach Ready; CrashLoopBackOff right after rollout rev43 (PR #482).
topology
deployment → pod → configmap; checkout-db ruled out.
root cause
rev43 removed the required ConfigMap key DB_HOST.
tools
ocp.rollout_status / pod_status / events / pod_logs · git.diff_range + get_pr_for_commit
Discriminator: the pod is never Ready (config panic at boot) — distinct from a resource-limit OOMKill, which flaps Ready. Closes mitigated.
INC-7702

fraud-scoring egress blocked by an ACL

Firewall / Security✓ golden
symptom
Egress calls fail cleanly (policy denies, not packet loss) after an ACL tightening.
topology
service → egress via firewall rule FW-EGR-118.
root cause
CHG-3311 tightened FW-EGR-118 (egress ACL).
tools
splunk.search_fw_denies / fetch_logs · servicenow.query_change_log
Discriminator: clean policy denies (vs. retransmits) rule out a link fault; a premature_write attempt exercises the gate block.
INC-9100

checkout-api pool saturation, empty change log

No-change / Saturation✓ golden
symptom
DB-pool saturation under a 3.4× organic traffic surge — and the change log is empty.
topology
service → database connection pool.
root cause
organic surge — no causal change to revert.
tools
prometheus.range_query / fetch_metrics · servicenow.find_recent_changes (returns none)
Discriminator: a phantom-change rival is ruled out by the empty change log; with nothing to revert, the correct terminal outcome is mitigated, not resolved.
INC-8801

order-processor consumer lag after a deploy

Messaging✓ golden
symptom
Consumer lag on the orders.events topic climbs to 61k; fulfilment SLA breaches.
topology
service consumes_from message_queue; a consumer deploy (CHG-55) shipped a slower handler.
root cause
change_event:chg-55 — consumer processing rate fell below ingress.
tools
prometheus.fetch_metrics (consumer_lag, dlq_depth, throughput) · git.diff_range · servicenow.list_related_incidents
Discriminator vs. broker/producer: the DLQ stays empty and producer throughput is steady — the backlog is purely consumer-side, ruling out a rebalance or a poison flood.
INC-8900

checkout pod evicted — noisy neighbour

Infra✓ golden
symptom
A tier-1 pod is evicted ("node low on resource: memory") on node-prod-17.
topology
pod runs_on host; an unbounded etl-nightly batch job is co-scheduled on the same host.
root cause
batch_job:etl-nightly — starved the node; kubelet evicted the lowest-priority tenant.
tools
ocp.events (eviction) / pod_status · prometheus.fetch_metrics (HOST mem 98% vs POD mem 55%)
Discriminator vs. app memory leak: the host memory is pegged while the victim pod's own memory was moderate right up to eviction — a co-tenancy fault, not the app leaking.
