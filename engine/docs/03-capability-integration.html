Capability Integration Guide

iw reaches every tool through one seam: an Adapter that pairs a binding-agnostic normalize(raw) → Operation[] with its data descriptors (provider, intents, effect, binding). Today every adapter is backed by a mock transport; this guide is the contract for wiring the real vendor — per capability, what to ask the owning team for.

01The integration model

A capability is a pure pair. The normalize half is deterministic and binding-agnostic — it maps a raw vendor payload to typed graph operations and never changes when the transport changes. The transport half (the one side-effecting seam) lives behind a Source and is swapped from mock to real without touching normalize.

class Adapter(Protocol):
    provider: str            # "servicenow", "prometheus", ...
    intents:  frozenset[str] # the verbs the planner may emit in `calls`
    effect:   Effect         # READ | WRITE  (WRITE is human-gated)
    binding:  Binding        # MCP | REST | A2A  (how the transport reaches it)
    def normalize(self, raw: dict) -> list[Operation]: ...

Binding · MCP

The vendor (or a wrapper) exposes a Model Context Protocol server; iw calls tools by name. Preferred for rich, evolving vendor APIs.

Binding · REST

A plain HTTP/JSON endpoint. Used where the vendor API is simple and stable (metrics, git).

Binding · A2A

Agent-to-agent — a governed action handed to another system that executes it. Used for the WRITE remediation, so the effect is owned + auditable there.

Gate-first serving. layer.serve() resolves → gates → fetches → normalizes. A WRITE with no approved gate returns before any transport call, so a side-effect can never precede human approval.

03Per-capability — what to request

servicenow

MCPREAD

The incident origin + change/CMDB context. Surfaces the incident record, firing alerts, recent changes, and co-firing related incidents.

Request from the ServiceNow / ITSM team
  • Instance URL + an OAuth (client-credentials) integration user, read scope on incident, change_request, cmdb_ci*, em_alert.
  • Table API read access (or the ServiceNow MCP server endpoint if one is provided).
  • The field mapping for change→CI (cmdb_ci, u_commit_sha) and related-incident correlation.

prometheus

RESTREAD

RED/USE metrics + firing alerts — the quantitative symptom and its recovery.

Request from the Observability team
  • The Prometheus / Thanos query endpoint (/api/v1/query, /query_range) + Alertmanager /api/v2/alerts.
  • A read token/bearer, and the metric names/labels for the RED (rate/errors/latency p50/p95/p99) and USE (cpu/mem/disk/net util + saturation, conn-pool, consumer-lag, dlq-depth) families.

appd

MCPREAD

Application topology + the JDBC/HTTP exit-call boundary that discriminates a callee-bound tail from a caller code path.

Request from the APM (AppDynamics) team
  • Controller URL + an API client (read), the app/tier/BT identifiers, and access to snapshots, flow maps, BT health, and health-rule violations.
  • The AppDynamics MCP endpoint if available, else REST controller API v2 access.

cmdb

MCPREAD

The declared dependency spine — seeds the topology and answers impact analysis.

Request from the CMDB / Platform team
  • Read access to CI classes + relationships (Depends on / Runs on / Connects to), CI attributes, and impact-analysis.
  • The discovery-source + last-discovered timestamp per CI (feeds node provenance).

splunk

MCPREAD

Logs, error signatures, and firewall denies — the textual evidence.

Request from the Logging / SIEM team
  • Search head URL + a service account with read on the relevant indexes; saved-search or MCP tool access for error-signature top-K, transaction traces, and firewall-deny search.
  • The index/sourcetype map for app errors and firewall logs.

ocp

MCPREAD

OpenShift / Kubernetes rollout, pod, and event state — deployment + infra faults.

Request from the Container Platform team
  • A read-only ServiceAccount / kubeconfig (get/list/watch on pods, events, deployments, replicasets) scoped to the relevant namespaces.
  • Access to pod logs + rollout history; the OpenShift MCP endpoint if provided.

git

RESTREAD

Diffs, blame, and PR metadata — resolves a change to the exact file:line.

Request from the SCM (GitHub/GitLab) admins
  • A read (contents + PRs) token/app scoped to the relevant repos; the REST API base URL.
  • The repo↔service mapping so a change_event resolves to the right repository.

artifactory

MCPREAD

Build → artifact → promotion lineage (the supply-chain spine).

Request from the Build / Release team
  • Read access to builds, artifacts by digest, and promotion history (AQL); the registry base URL + a read token.

remediation

A2AWRITE · human-gated

The one side-effecting capability. The LLM proposes a concrete, reversible apply_remediation action; the run suspends at the human approval gate carrying the action + its evidence chain; on approval the action is handed to the executing system.

Request from the Change / Automation team
  • An A2A endpoint (or runbook-automation / ServiceNow-Orchestration hook) that accepts a typed action and returns a recorded effect — never executed by iw directly.
  • The approved catalogue of reversible actions (rollback deploy, recreate index, revert ACL, reschedule job, scale pool) + the authorization/audit contract.
  • Confirmation that every action is idempotent + reversible, and that execution is logged on the vendor side.
Boundary: iw entering credentials or executing a financial/destructive action directly is out of scope by design. iw proposes + records; the A2A executor + the human own the consequence.

04Mock → real migration path

The migration is per-capability and non-breaking, because normalize is transport-agnostic:

StepWhat changes
1 · Keep normalizeThe normalize(raw) for each adapter is already written + tested against mock payloads. It does not change.
2 · Swap the SourceReplace MockSource / ScenarioSource with the real transport (McpSource, RestSource, or a routed mix) for that provider only.
3 · Map the payloadIf the real payload shape differs from the mock, adjust that adapter's normalize field-reads — the one place a vendor quirk lives.
4 · VerifyThe golden e2e suite stays on mocks (deterministic CI); a gated live smoke run exercises the real transport per provider.
Do one capability at a time. Because each adapter is isolated behind the same seam, ServiceNow can go live while the rest stay mocked — the engine, reducer, and every other capability are unaffected.
