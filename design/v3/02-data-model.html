02 · Data Model

The Entity-Assertion model, at class level: one Assertion atom with five temporal species and channel-keyed belief; facts, events and node props as read views over a single store; the semantic dictionary and closed registry that make the vocabulary governable; the hypothesis record whose belief is earned, never asserted; and the airlock that admits the unknown without ever erasing it. Every field name, signature and id format on this page is copied from the tree — open the source and find it.

01One record, one map

The old model gave every kind of knowledge its own container — node properties, facts, events — and each container leaked: props had no source, metric names encoded their units, the same observation lived in three shapes. The v3 ruling collapses all of it into one record: everything the run learns is a typed Assertion about an entity or an edge, and everything else — the facts view, the events view, node props, evidence edges — is derived from that one collection. Entities and edges come from a closed registry; names come from a semantic dictionary; belief comes from arithmetic the engine owns.

02Entity — classification, identity keys, aliases

An entity is a Node whose type comes from the closed catalog and whose id is a pure function of its identity keys. There is no bespoke Python subclass per type — each type is a declarative NodeSpec, one source of truth for the LLM's allowed-types schema and the reducer's validation, so the two can never drift.

# domain/spec.py
@dataclass(frozen=True)
class NodeSpec:
    type: NodeType
    tier: str                                  # "L0".."L6" | "signal"
    identity_keys: tuple[str, ...]             # props that form the upsert identity
    static_props: tuple[str, ...] = ()         # allowed IMMUTABLE/identity props (time-varying → Fact)
    fact_predicates: tuple[str, ...] = ()      # legacy — dictionary applies_to is the live authority
    event_types: tuple[str, ...] = ()          # feeds the dictionary's derived event entries
    discriminator: str = ""                    # rule to pick this type vs siblings (LLM-facing)

Note the service row: repo, app_id, sys_id and k8s_workload are the identity backbone — the per-tool foreign keys the ServiceNow adapter resolves off the incident's CMDB CI. They are not inert cargo: the resolver lifts them into an aliases {scheme: id} block and the graph indexes them, which is what kills split-brain twins (§11). The rule that keeps props honest is R-G5: a static prop is identity or immutable only; anything time-varying (image, replica counts, tier) is a windowed assertion, never a prop.

Deterministic identity — and rejection over degeneracy

# domain/registry.py
def _slug(v: object) -> str:
    # P5 identity hardening: `_` collapses like space/`/`, so
    # `payments_api` == `payments-api` == "Payments API" — cross-tool
    # spellings of one name stop minting split-brain twins.
    return str(v).strip().replace(" ", "-").replace("/", "-").replace("_", "-").lower()

def node_id(ntype: NodeType, props: dict) -> str:
    """Idempotent identity — same type + identity_keys → same id (upsert key)."""
    spec = NODE_SPECS[ntype]
    keys = spec.identity_keys or tuple(sorted(props.keys()))
    parts = [_slug(props.get(k, "")) for k in keys]
    return f"{ntype.value}:" + "|".join(parts)

Run for real: node_id(SERVICE, {"service_name": "payments_api", "env": "prod"}) returns service:payments-api|prod — the underscore spelling and the hyphen spelling converge on one entity. A missing identity key is a journaled Rejection (“refusing a degenerate id”), never a guessed or empty id — unless the arrival carries a backbone credential, in which case the provisional-identity lane opens (§11).

A Node record's real fields (seen at every mint site): id · type · props · aliases · provisional · source · created_by. Since the P6 store-flip, props is a materialized read view: every prop is a DECLARED-channel assertion in the one store, and the dict on the record is rebuilt from those assertions on each upsert (§04). source rides the record so each prop's declaration is attributed to the adapter that supplied it — per-prop provenance is real, not inferred.

03The Assertion atom — five species, one envelope

One record answers five different questions, distinguished only by their time shape: what is this (identity) · what do we know about it (descriptor) · what is true over time (state) · what did we measure (reading) · what happened (event). Everything else — provenance, belief, lifecycle — is one shared envelope. This kills the Fact-era trichotomy that made props unsourced and forced red_rate to encode rpm-vs-baseline in its name: window and stat are now fields, not name fragments.

domain/assertion.py — class Assertion(BaseModel) · extra="forbid" · the full envelope

Subject & semantics

  • id: str — fact:/evt:/prop: namespaces (§06)
  • subject_ref: str — a NodeId or EdgeId (edge-borne RED is reachable)
  • name: str — dictionary-canonical spelling (§05)
  • value: AssertionValue = bool | int | float | str | dict | list | datetime | None
  • unit: str | None · where: str | None

Time — the species axis

  • species: Species — the five below
  • valid_from / valid_to: datetime | None — the truth window; valid_to=None = still true
  • observed_at: datetime | None — transaction time (when we learned it)
  • occurred_at: datetime | None — EVENT only: when it happened in the world
  • stat: Stat | None + window: Window | None — READING only

Provenance

  • source: Source — one of 12 (prometheus … bigpanda, llm, human, engine)
  • source_native_name: str | None — the vendor's own spelling, surviving canonicalization
  • evidence: list[EvidenceRef] — {kind, ref, label}: metric_query · trace_id · log_link · snapshot · diff · blame · ci_record · url
  • created_by: int — the journal seq that minted it (lineage)

Belief — keyed on channel, not source

  • channel: Channel — measured · inferred · declared · engine
  • confidence: Confidence | None — INFERRED channel only; value + mandatory basis
  • source_reliability: float | None (ge=0, le=1) — MEASURED / DECLARED / ENGINE
  • exactly one is meaningful; which is fixed by the channel (R-C4)

Lifecycle

  • state: FactState — active · superseded · retracted
  • supersedes: str | None — the id this replaces (never mutate — supersede)
  • invalidated_by: str | None — what proved a retracted record wrong
  • is_open property — valid_to is None and state == ACTIVE (the supersession-scan predicate)

The airlock flag

  • provisional: bool = False — True for knowledge the airlock admitted rather than the closed vocabulary: a quarantined name or an off-shape reading (§10)
  • rendered dimly, counted toward promotion, never silently erased
_window_ok valid_to < valid_from (both set) raises — no inverted truth window can exist.
_time_shape identity: no observed_at, no validity window (write-once, timeless) · every other species requires observed_at — except a DECLARED descriptor (a node-prop declaration is asserted configuration truth, minted with no wall-clock stamp so journal replay stays deterministic) · reading requires both stat and window · event requires occurred_at, and occurred_at is EVENT-only.
_belief_channel identity carries neither belief field (asserted truth, not belief) · event carries at most one (P1a-lenient) · inferred must carry confidence and never a reliability · measured / declared / engine must carry source_reliability and never a confidence.

The three guards in the panel above are real model_validator(mode="after") methods — _window_ok, _time_shape, _belief_channel — and they raise at construction time. On the planner path the reducer catches the raise into a journaled Rejection ("invalid fact: …") instead of crashing the run (INV-7); on the adapter path a violating emission never leaves normalize().

A reading's Window is itself validated: a point at xor a range start+end (both required, end ≥ start) — so a reading's evidential window is never ambiguous. And Confidence can never be a naked float:

Channel-keyed belief

Belief used to hang off Source.LLM as an identity special-case. The atom moves it onto an explicit channel, chosen so the Fact-era discipline round-trips exactly — channel == INFERRED iff the source is the reasoning model:

# domain/assertion.py
_SOURCE_CHANNEL: dict[Source, Channel] = {
    Source.LLM: Channel.INFERRED,
    Source.ENGINE: Channel.ENGINE,
}
def channel_for_source(source: Source) -> Channel:
    """LLM → inferred (confidence); engine → engine;
    every directly-observing tool/human → measured (reliability)."""
    return _SOURCE_CHANNEL.get(source, Channel.MEASURED)

Channel.DECLARED is deliberately absent from that map: it is minted in exactly one place — Graph._declare_prop, the node-prop seam (§04) — so a converted Fact can never leak into the props view. The LLM never emits a numeric belief at all: it emits the coarse ConfidenceLevel rubric (low / med / high), which the reducer maps through the playbook's confidence_band tunable (defaults 0.3 / 0.6 / 0.9) into a cited Confidence — the R-C4 discipline that kills “0.9 everywhere”.

Op-boundary vs store — an honest caveat. Adapters emit the atom op with the full reading shape — species=reading, stat=gauge, window={at: …} — and the validators demand that shape at the op boundary (shape_violation checks it, §05). But today's reducer materializes non-event assertions as Fact records, and the store seam (shim.assertion_of_fact) re-derives species from the predicate with no reading shape available — so stored assertions carry species identity / descriptor / state / event only, and stat/window live at the op level. The shim docstring owns the choice: the species is “never surfaced in any rendered view, so a reclassification cannot move a byte in the bundle.” When in doubt the boundary test says state — the cheap direction: a window you never query costs nothing.

04Facts, events, props — read views over one store

The P6 store-flip: Graph keeps one assertion collection, and the three legacy containers become cached read views over it. There is no dual write left to drift — to_dict() persists {"nodes", "edges", "assertions", "remaps"} and nothing else (GRAPH_SCHEMA_VERSION = 2). The flip landed with goldens byte-identical because the shim converters are exact inverses.

No leaks channel_for_source() never returns DECLARED, so a converted Fact can never appear in the props view; the props channel is minted only by _declare_prop. The converters (domain/shim.py) are an exact-inverse pair — every field round-trips byte-identically.

Node props are DECLARED assertions

upsert_node turns every arriving prop into one assertion via _declare_prop(nid, ntype, key, value, source, seq), then rebuilds the record's props dict from the view — never dual-authored. The rules, exactly as coded:

  • Assertion id prop:<node>:<key> — a re-declaration replaces in place: props are static per R-G5, so a changed value is a correction, not history (time-varying values belong in windowed assertions).
  • Identity-key props mint species=identity with no belief field; all others mint species=descriptor with source_reliability=1.0. Channel is always declared; source defaults to engine for unsourced arrivals.
  • No observed_at — the _time_shape carve-out: replay must never stamp a wall clock, so prop declarations are timeless and journal replay stays deterministic.
  • Merge semantics from the dict era survive: a first mint lands every key (None included); a later upsert's non-None values win, its None values never override; an unchanged value is a no-op, so the first declarer's provenance survives (write-once flavor, symmetric with the alias index).

Never mutate — supersede or retract

# graph/graph.py — add_fact: the supersession seam (and its clamp)
if fact.supersedes and fact.supersedes in self.assertions:
    old = self.assertions[fact.supersedes]
    # CLAMP, not reject: a back-dated correction (new.valid_from < old.valid_from)
    # would otherwise persist an inverted window. Clamping to old.valid_from yields
    # a zero-length window — "no instant at which the old value was the truth".
    # add_fact is the single mutation seam for BOTH live fold and journal replay:
    # it has no rejection channel, and raising could brick crash-resume replay.
    closed_at = (max(fact.valid_from, old.valid_from)
                 if old.valid_from is not None else fact.valid_from)
    self.assertions[fact.supersedes] = old.model_copy(
        update={"valid_to": closed_at, "state": FactState.SUPERSEDED})
self.assertions[fact.id] = assertion_of_fact(fact)

Retraction is the other lifecycle exit: retract_fact / retract_event / retract_edge flip state=retracted — a tombstone for an observation that was wrong, distinct from superseded (was true, then replaced). Nothing is ever deleted. One asymmetry worth knowing: retract_fact takes no invalidated_by (the Fact record has no such field — the cause lives in the journaled Retraction record), while events and edges carry it on the record.

Derived transition events — the engine authors the flip

One more writer feeds the events view: _derive_transition_events (graph/reducer.py, P6 step 5). When a boolean STATE fact flips — False/absent → True, or True → False — the reducer derives the matching <name>_started / <name>_cleared EVENT in the engine, so adapters and scenarios stop dual-authoring an occurrence twin for every boolean state they report. The discipline, exactly as coded: dictionary-known transition names only (the engine never fabricates a quarantine spelling it invented itself); dedup by event id against graph + batch — an authored twin wins; provisional (airlocked) facts never derive; and the derived events ride the delta (Materialized.events → PhaseResult → journal), so replay reproduces them bit-for-bit with zero engine-state dependence. The prior value is read from the superseded assertion (or the open same-predicate fact), so a re-asserted True or a never-started False derives nothing. Gated by tunables.derive_transitions — default off: the shipped scenarios still author their transitions, and several deliberately model stacks that don't emit them; flipping it on plus de-dual-authoring the scenario twins is the recorded P7 follow-up.

Point-in-time truth — INV-5

# graph/graph.py — facts_valid_at (principle 8)
return [f for f in self.facts.values()

SUPERSEDED facts are included — a superseded fact is still the truth for every instant inside its now-closed window. Excluding them broke as-of-incident-start reconstruction the moment any fact was superseded (the finding that named INV-5). Only RETRACTED — never true at any instant — is excluded from history.

Seven tools spell “error rate” seven ways. The dictionary (domain/dictionary.py) is ONE table keyed by canonical name — the authority the reducer canonicalizes and validates against. The vendor's spelling is never lost: it survives on every assertion's source_native_name, and — crucially — fact identity is keyed on the native name, so a vocabulary rename never moves an id.

The shape lane, precisely (shape_violation): a unit mismatch fires only when both the op and the entry declare a unit and they differ — “a twin's None/omitted unit never drops a fact”; a claimed reading without stat+window fires; a stat mismatch is deliberately unchecked today, because the P1a/P1b adapters stamp stat=gauge as a compat default on metrics whose entries declare rate/ratio/percentile — it becomes checkable when adapters emit true stats. Events return None from the shape check (they carry no reading shape); unknown names go to the name lane instead.

One verified sharp edge. The dictionary assembly comment says “fact names win any (nonexistent) collision” — but the collision exists: degraded is both a STATE fact on service and a declared CLUSTER event type (nodes/platform.py, event_types=("upgraded", "degraded")). setdefault keeps the fact entry, so a cluster degraded event fails applies_to_ok and is rejected despite the registry declaring it legal. Known divergence; the code is the behavior described here.

06The closed registry — classify, never invent

Principle 4: the LLM picks a member of a closed vocabulary; it never mints a label. NodeType.GENERIC_CI is the single escape hatch. The registry (domain/registry.py) binds the enums to their specs and refuses to import if any member lacks one — the closure guarantee R-G1, checked in three places (registry.py, nodes/__init__.py, edges/__init__.py) so an incomplete tier module fails loudly at the source:

# domain/registry.py — evaluated at import
_missing_nodes = [t.value for t in NodeType if t not in NODE_SPECS]
_missing_edges = [t.value for t in EdgeType if t not in EDGE_SPECS]
if _missing_nodes:
    raise RuntimeError(f"registry incomplete — NodeTypes without a spec: {_missing_nodes}")
if _missing_edges:
    raise RuntimeError(f"registry incomplete — EdgeTypes without a spec: {_missing_edges}")

Because the LLM-facing catalog is rendered from these same specs (plus dictionary.fact_names_for), an off-catalog op is by construction exactly an op the reducer will reject — the schema the model sees and the validation it hits cannot drift apart.

Three consequences worth naming: add_edge is idempotent by id (type + endpoints + origin are the id); a remap can re-mint edge ids deterministically because endpoints are embedded in them; and the id namespaces (fact: sha1-16, evt: sha1-16, edge:…, prop:…, hyp:…) are disjoint, which is what lets the fold dispatch a retraction by store membership without ambiguity.

08Edges — declared spine, inferred causality

Causation is a separate, refutable layer — never mixed into structure. Every edge declares origin ∈ {declared, discovered, inferred} (R-G8): declared = CMDB/IaC, treated as truth; discovered = telemetry, an observation; inferred = a hypothesis's claim, confidence + evidence mandatory. The graph is a MultiDiGraph, so a structural depends_on and an inferred caused_by between the same pair coexist under distinct ids.

Why the spine is “governed”: every belief-relevant traversal — Graph.structural_distances, and the P7 tool surface built on it — crosses only ACTIVE edges whose origin is not inferred, and never enters a hypothesis node. Topological specificity must come from observed structure, so a hypothesis's own causal claims can never shorten the distance of its own evidence — the anti-bootstrapping rule the earned-belief arithmetic (§09) depends on. Edge records carry the same lifecycle as facts (state, invalidated_by, valid_to on retraction): a refuted caused_by tombstones exactly as a wrong fact does. And because the reducer's known() learns edge ids, an assertion may take an edge as its subject — a discovered calls edge can carry its own RED readings.

09The Hypothesis record — belief is earned

A hypothesis is never a floating string: it is a typed record holding BOTH supporting and refuting fact-id lists (anti-confirmation-bias, principle 10) and a typed causal chain. It lives twice on purpose — a store record ranked by lifecycle status then earned belief, and a graph node hyp:<hid> so causal edges can reference it. The naming is deliberate: this is a hypothesis store (hypothesis/store.py) — its former name “Ledger” is retired (owner directive 2026-07-22: “remove ledger, that is what journal is for”), because it blurred the role split that matters: the append-only journal is the audit/review + replay/resume record, while the store is a mutable ranked projection the planner actually reads — its ranked() summary rides every PlanContext, alongside the graph's focus slice. The bundle exports it under the key hypotheses; the session streams hypotheses_delta events.

    refuting_facts: list[str]       # FactIds → the derived REFUTES edges

Store semantics that make a live planner safe: a re-CREATE of a hid whose status is terminal (refuted / confirmed / superseded — _TERMINAL) is a silent no-op: a reached verdict is indestructible evidence and can never be reset to proposed by a model re-proposing the same id. A re-CREATE of a live hid is a merge, never an overwrite: status and created_by preserved, evidence lists grow by sorted set-union, the chain appends. (Precision the docstrings don't volunteer: indestructibility covers the CREATE path — an update delta can still move any status; there is no engine transition table. That is exactly why the promotion gate below, not the store, is the authority that matters.)

The blend score = round((prior_weight·band + Σ w_sup) / (prior_weight + Σ w_sup + Σ w_ref), 4) — the LLM's band enters as a pseudo-observation of mass prior_weight (1.0); supporting evidence pulls toward 1, refuting toward 0. No resolvable evidence ⇒ exactly the band.
INV-8 gate promotion_ok: earned score(lead) ≥ confidence_gate 0.8 ∧ zero alive unrefuted rivals (any band — Popperian: rivals must be refuted, not out-scored; an unrefuted 0.75 rival under a 0.8 gate still blocks) ∧ the delta 0.15 floor. Promotion is the ENGINE's decision — an LLM-set confirmed status grants no bypass.
# hypothesis/store.py — promotion_ok, verbatim
alive = self.alive()
if not alive:
    return False
lead = alive[0]
if self.score(lead) < tunables.confidence_gate:
    return False
rivals = alive[1:]   # ALL alive unrefuted rivals block — no band filter
if rivals:
    return False
runner = self.score(alive[1]) if len(alive) > 1 else 0.0
return (self.score(lead) - runner) >= tunables.delta

Read the code, not the docstring's “beats the runner-up by delta”: because ANY rival returns False first, the runner-up expression only executes when the leader is alone — runner is always 0.0, so under the strengthened rival rule delta acts as a second absolute floor on a sole survivor, not a live margin race.

Three more precisions from the code. Ranking is (status, then score): ranked() sorts by the lifecycle ladder first (_STATUS_RANK: confirmed 5 · supported 4 · investigating 3 · proposed 2 · refuted 0 · superseded −1), then by the earned score — so a refuted hypothesis stays listed (it is evidence) but can never outrank a live one. Scoring is a bound, pure projection: the engine calls bind_scoring(graph, tunables, anomaly_ref) to attach the arithmetic's context; scoring is query-time and side-effect-free, so a bound store needs no cache invalidation — and an unbound store (bare unit tests, raw rebuild) scores every hypothesis at exactly its band. The proximity anchor is data, not a parameter: belief.onset_of takes the anomaly node's earliest non-retracted onset_value fact (deterministic by (valid_from, id)) as symptom onset and measures each evidence fact's valid_from against it; until an onset is framed, proximity is neutral (1.0) — absence of an anchor is never a penalty. When no explicit ref is bound, find_anomaly falls back to the first Anomaly node in creation order, which journal replay reproduces.

10The airlock — nothing observed is erased

The audits' worst finding about the old reducer: rejected observations were erased — “they never existed.” The airlock replaces reject-and-erase with quarantine-and-record: unknowns land provisional, dimmed and counted, never silent, never gone. Four lanes, each with an exact mechanism:

The four airlock lanes (graph/reducer.py · domain/dictionary.py · domain/registry.py)
1 · name Unknown name → lands under quarantine_name(source, native) = x.<source>.<native>, provisional=True, applies_to skipped. Deterministic — repeats COUNT toward promotion; collision-free — no canonical starts with x. and the reserved no_evidence: engine prefix cannot be spoofed through it. Promotion is a human core-registry edit. Referential integrity is NOT relaxed: an unknown subject still rejects.
2 · shape Known name, invalid shape (unit mismatch; a claimed reading without stat+window) → lands provisional=True plus a journaled Rejection notice: “shape quarantine '<name>': … — landed provisional.” Never silently accepted (the mismatch is on record, feeding the planner) and never erased (the observation survives, dimmed).
3 · structure generic_ci becomes placeable: edge_airlocked() lets it substitute for either endpoint of a STRUCTURAL edge iff the non-generic endpoint is already legal on its side (both-generic allowed) — it stands in for half of an existing pair, never mints new pair semantics. And blamable: caused_by → generic_ci is a declared pair. Every airlocked edge is provisional; origin is FORCED to discovered (except caused_by, which stays inferred per spec); its confidence is multiplied by discovery_penalty 0.75 with the basis suffix [provisional: generic_ci endpoint] — admitted, never at full weight.
4 · null result NoEvidence (R-P2) — but error ≠ evidence: an op naming an intent whose last boundary outcome was error or blocked is REJECTED — “an error carries no evidentiary weight; only a clean-empty read can become null evidence.” An accepted one materializes as an engine fact: predicate no_evidence:<intent>, source=engine, source_reliability=1.0, subject = the resolved scope else the anomaly node.
capability boundary outcomes: data · empty · error · blocked — the code literal is "empty" (the honest clean-empty), and error ≠ empty is the honesty line (depth: doc 05)

The grammar of correction — Retract, Merge, Retype

retract — R-J3

Target must EXIST (graph or earlier in this batch) else rejection. Becomes a Retraction record on the delta; the fold applies retractions after additions, dispatching by id namespace. State flips to retracted; the record survives as evidence of what was once believed.

merge — R-J5

Provisional → canonical only. Rejected if: either id unknown · already the same entity · the source is canonical (“canonical entities never merge”) · the target is itself provisional (“graduate it first”) · cross-type. Success = a journaled Remap(kind="merge"); the fold's remap_id re-homes every reference.

retype — the hatch graduates

generic_ci only — “typed identity never re-keys.” Never to generic_ci/hypothesis; must supply the new type's identity keys; the minted id must not already exist (“retype mints, it does not fold”). ci_id/class_hint survive as provenance; the old id stays resolvable forever.

graph/reducer.py — the AddNode resolution lanes, in branch order (pass 1)
A · upsertThe computed id already exists (graph or batch) → plain upsert; provisional twins bound to this arrival's credentials auto-merge in (late alias binding).
B · resolveThe split-brain kill: exactly ONE same-type canonical entity is bound to one of the arrival's credentials → fold onto it, never mint a twin. The arrival's identity-key props are DROPPED (“identity is write-once on the canonical; the alias, not the key, is what unified them”); a journaled Remap(kind="resolve") makes the redirect permanent, so a later phase citing the would-be twin id still lands on the canonical after any replay.
C · mintId computable, zero or ≥2 canonical hits → fresh mint. Two hits are never guessed between — ambiguity rejects instead (below).
D · accumulateNo computable id, exactly one provisional twin holds the credential → observations accumulate on the twin (still provisional).
E · provisionalNo id, no hits, but a credential exists → mint {type}:~{scheme}:{slug}, provisional=True — instead of rejecting; a later Merge (auto or explicit) graduates it.
rejectMissing identity key(s) and no credential — “refusing a degenerate id”; or ≥2 same-type alias hits — “ambiguous alias resolution”. Never a degenerate type: id, never a guess.

Two guarantees hold the index honest. First binding wins: the graph binds an alias key with setdefault; a later claim for a different node is a journaled contradiction on the rejections channel (“alias contradiction: … first binding wins — node materialized, alias claim recorded”) — never a silent rebind that would re-attribute every future observation. Old ids live forever: a merge/retype/resolve applies through Graph.remap_id, which chain-compresses the id_remaps table (the old id becomes an alias, resolvable forever), re-keys prop: and edge: ids (their ids embed the node id; identical-edge collisions collapse first-writer-wins), and rewrites subject_ref in one pass — fact and event ids are minted once and never move, so provenance ordering, supersession chains and the hypothesis store's fact-id refs stay byte-stable through any identity graduation.

12Invariants — what must hold, and what breaks

Four numbered invariants are named in code comments at their exact enforcement points (INV-5, INV-7, INV-8, INV-9 — grep the tree); three more are design ids whose enforcement now exists in code (INV-1, INV-2, INV-11). Alongside them stand the standing design rules (R-G*, R-C*, R-J*, R-P*, R-K*) from design/v3/DESIGN.md. Each row: the rule, what it guarantees, and what breaks the day it is violated.

No side effect on a live system ever precedes consent; the journal proves the question and the answer. Silent production writes on a model's say-so.

The one-sentence summary. One typed assertion atom, validated at the boundary, canonicalized by a dictionary, attached to entities from a closed registry, folded through a single seam into views the whole system shares — with belief earned by arithmetic, unknowns quarantined instead of erased, and every rule above enforceable because there is exactly one place to enforce it.
