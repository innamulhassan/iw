What ships each sprint

SprintMilestoneWhat ships
S1M0Repo + skeleton: engine graph, capability registry interface, Postgres/Mongo schema, one read-only adapter (AppD), shadow-mode harness.
S2M0In-phase loop + checkpointer + govern() + append-only audit + the console shell. Exit: an assisted run end-to-end on a read-only capability in shadow.
S3M1 · assistedAssess phase live — incident-source + topology + telemetry + change-history + similar-incidents adapters; AssessResult. Amelia begins running alongside on the same incidents (for comparison).
S4M1 · assistedRoot cause phase — metrics/logs/traces adapters; ranked candidates + confidence; the console chat + clickable evidence.
S5M1 · assistedFull console (3 panes) + HITL approval surface + timeline/audit views. Exit: triages real incidents read-only on a pilot slice; operators trust it.
S6M2 · augmentedRemediation — the first gated write (a mitigate / reverse action) + idempotency + the approval card + the rollback plan. (Hardening baseline must be green.)
S7M2 · augmentedmore techniques (failover · reroute · restart · scale) + escalation; error_handler (retry / escalate / manual) + on-failure + partial / blocked handling.
S8M2 · augmentedVerify & close + synthetic replay; the operator-close flow. Exit: gated fixes execute safely with rollback, exactly-once.
S9M3 · augmentedMulti-app / cross-layer topology + incident relationships (parent/child/related/chained) + alert→incident ingestion / correlation.
S10M3 · augmentedA2A specialist agents (DB, network) + remote MCP; Amelia registered as an A2A capability, its actions gated; sub-agent governance.
S11M3 · augmentedMore capabilities + domains; playbook studio (author · validate · publish); registry/governance admin. Exit: handles a multi-app incident; N playbooks live.
S12M4 · autonomousAutonomy dial for the first proven low-risk action (per phase × node-type × severity); the feedback → policy loop (promote/demote).
S13M4 · autonomousThe autonomy dashboard + metrics; expand autonomous actions; ramp Amelia down on the proven slices.
S14M4 · autonomousBroaden autonomy on proven actions; complete the Amelia ramp-down for covered domains. Exit: a measured share of low-risk actions run autonomously with zero regressions, all reversible.
Parallel track — bank-grade hardening (gates production writes). Secrets / KMS, encryption at-rest + in-transit, audit immutability, failure / DR (RTO / RPO targets), and the test strategy (dry-run / shadow) are designed in S1–S2 and in place before M2 (S6) — no write capability reaches production without them. (Scoped in the Design doc hardening track.)
Why phased. Value lands at M1 (faster diagnosis) before any write risk. Each milestone is gated, audited, reversible. Autonomy (M4) is earned narrowly only after M1–M3 prove the engine. Amelia is never cut over big-bang — it runs alongside, is wrapped as a governed capability, and is dialled down only as in-house actions prove themselves on each slice.
